# 模擬定位測試器

這是一個 Android Kotlin / Jetpack Compose 測試工具，用於測試你自己的定位應用程式在模擬定位下的行為。

不得用於第三方應用程式、遊戲作弊、控制第三方應用程式、點擊注入、規避平台偵測、規避服務限制，或任何違反服務條款、法律、平台政策的用途。

## 功能

- 使用 `LocationManager.GPS_PROVIDER` 與 `LocationManager.NETWORK_PROVIDER` 建立模擬定位提供者。
- 使用 `LocationManager.addTestProvider()`、`LocationManager.setTestProviderEnabled()`、`LocationManager.setTestProviderLocation()` 同時推送 GPS 與網路位置。
- 應用程式啟動時請求定位權限，Android 12 以上請選擇精確位置。
- 可輸入起始緯度、起始經度、精度。
- 手動搖桿、懸浮搖桿、路徑巡航與區域巡航速度都使用 km/h 顯示。
- 搖桿最大速度可調整，範圍 1 到 100 km/h，預設 5 km/h。
- 路徑巡航與區域巡航速度可調整，範圍 1 到 100 km/h，預設 5 km/h。
- 每次成功推送模擬定位後會保存最後位置，停止手動搖桿或懸浮搖桿不會重設經緯度。
- 可啟動「停留在最後位置」，每秒以 0 速度持續推送最後模擬座標。
- 主畫面虛擬搖桿可控制方向角與速度。
- 懸浮搖桿可顯示在你自己的應用程式上方，用於測試定位移動。
- 地圖繞行模式可切換「路徑選點」、「區域圈選」與「慢走到目的地」，並支援單點刪除與復原上一個點。
- 路徑巡航可選擇「從最近點開始」或「從第一點開始」，預設為「從最近點開始」。
- 區域巡航可用多邊形範圍產生「隨機巡航」或「蛇形掃描」路徑，範圍點可單點刪除。
- 慢走到目的地會從目前模擬位置依序走到一個或多個目的地，不會瞬移。
- 定位接收示範可分別顯示 `GPS_PROVIDER`、`NETWORK_PROVIDER`、`FusedLocationProviderClient` 收到的位置與是否為模擬定位。

## 啟用開發人員選項

1. 打開手機的「設定」。
2. 進入「關於手機」。
3. 連續點擊「版本號碼」7 次。
4. 輸入螢幕鎖定密碼後，系統會啟用「開發人員選項」。

不同品牌手機的選單名稱可能略有差異，例如「軟體資訊」或「版本資訊」。

## 選擇模擬位置應用程式

1. 打開「設定」。
2. 進入「系統」或「其他設定」。
3. 打開「開發人員選項」。
4. 找到「選取模擬位置應用程式」。
5. 選擇「模擬定位測試器」。

如果沒有完成這個設定，應用程式會顯示錯誤訊息，且無法推送模擬定位。

如果在清單中看不到本應用程式：

- 請確認安裝的是本專案產出的除錯 APK。
- 請重新安裝新版 APK 後，關閉並重新打開「開發人員選項」。
- 若手機仍顯示舊狀態，可先移除舊版再安裝：

```powershell
adb uninstall com.example.mocklocationtester
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

本專案已在 `AndroidManifest.xml` 宣告 `ACCESS_MOCK_LOCATION`，部分 Android 系統會依此決定是否讓應用程式出現在「模擬位置應用程式」清單中。

## 給應用程式定位權限

1. 第一次啟動應用程式時，系統會要求定位權限。
2. 請允許定位權限。
3. Android 12 以上請選擇「精確」位置。
4. 如果曾經拒絕權限，請到「設定」>「應用程式」>「模擬定位測試器」>「權限」>「位置」重新允許。

## 授權懸浮視窗

懸浮搖桿需要 Android 懸浮視窗權限。

1. 打開「模擬定位測試器」。
2. 在主畫面點擊「授權懸浮視窗權限」。
3. 系統會開啟 `Settings.ACTION_MANAGE_OVERLAY_PERMISSION` 對應頁面。
4. 允許本應用程式顯示在其他應用程式上層。
5. 返回「模擬定位測試器」。

Android 13 以上若系統詢問通知權限，建議允許，因為懸浮搖桿會以前景服務執行並顯示通知。

## 最後位置與起點

1. 每次成功推送模擬定位後，應用程式會保存最後一次模擬定位位置。
2. 下次啟動應用程式時，如果有最後位置，會預設載入最後位置並顯示「已載入上次停止位置」。
3. 「使用最後位置作為起點」會把起點設為上次模擬定位停下的位置。
4. 「使用目前手機位置作為起點」會透過手機真實定位取得目前位置，適合要重新從真實位置開始測試時使用。
5. 「清除最後位置」只會清掉保存的最後位置；清除後下次啟動才會回到預設座標。
6. 停止手動搖桿、停止巡航或關閉懸浮搖桿時，只會把速度設為 0，不會把經緯度重設成手機真實位置或輸入框舊座標。
7. 「停留在最後位置」會每秒持續推送最後模擬座標，速度為 0。
8. 「停止模擬定位」才會停止停留服務並移除測試定位供應器，系統才會回到手機真實定位。

## 使用手動搖桿

1. 進入「手動控制」分頁。
2. 輸入起始緯度、起始經度與精度。
3. 點擊「套用起始座標」。
4. 也可以點擊「使用最後位置作為起點」或「使用目前手機位置作為起點」。
5. 調整「搖桿最大速度」，範圍 1 到 100 km/h，預設 5 km/h。
6. 點擊「推送目前位置」可立即推送一次目前位置。
7. 點擊「開始走路模擬」後，會從畫面目前顯示的位置開始，不會自動改成手機真實位置。
8. 拖曳圓形虛擬搖桿控制移動方向與速度。
9. 搖桿方向代表方向角：北方 0 度、東方 90 度、南方 180 度、西方 270 度。
10. 搖桿離中心越遠，速度越高；放開搖桿後速度會變成 0。
11. 點擊「停止」後不再送出位置，畫面會保留最後模擬定位位置。

## 使用懸浮搖桿

1. 確認已授權定位權限、精確位置、模擬位置應用程式、懸浮視窗權限。
2. 在「手動控制」分頁選擇起點，可使用最後位置、目前手機位置，或手動輸入後點擊「套用起始座標」。
3. 調整「搖桿最大速度」，範圍 1 到 100 km/h，預設 5 km/h。
4. 點擊「啟動懸浮搖桿」。
5. 系統會啟動 `FloatingJoystickService`，並顯示通知「模擬定位搖桿執行中」。
6. 半透明圓形搖桿會顯示在畫面上方，可蓋在你自己的應用程式上方進行測試。
7. 拖曳搖桿可控制模擬定位移動方向與速度。
8. 放開搖桿後速度會變成 0。
9. 拖曳搖桿外的懸浮視窗區域可移動整個懸浮視窗位置。
10. 點擊懸浮視窗上的「關閉」或主畫面的「停止懸浮搖桿」只會關閉懸浮搖桿 UI。
11. 關閉懸浮搖桿後，應用程式會啟動「停留在最後位置」，每秒持續推送最後模擬座標，不會跳回手機真實位置。
12. 若要回到手機真實定位，請在主畫面點擊「停止模擬定位」。

懸浮搖桿只處理自己的視窗拖曳和自己的搖桿觸控，不實作控制第三方應用程式、點擊注入、繞過偵測或遊戲作弊功能。

## 使用地圖點選座標點

1. 進入「地圖繞行模式」分頁。
2. 上方是地圖，下方是可捲動控制面板。
3. 地圖會顯示目前模擬定位位置；第一次載入時若有最後位置，地圖會移到最後位置一次。
4. 點擊「路徑選點」後，在地圖上點一下會新增路徑點。
5. 點擊「區域圈選」後，在地圖上點一下會新增範圍點。
6. 點擊「新增目的地」後，在地圖上點一下會新增慢走目的地。
7. 每個路徑點與範圍點都會以編號標記顯示，刪除後編號會重新排列。
8. 點擊地圖標記後會顯示「第 X 個點」、「刪除此點」、「取消」。
9. 點擊「刪除此點」只會刪除該點，折線會同步更新。
10. 點擊「復原上一個點」只會移除目前模式的最後一點。
11. 點擊「清除全部路徑」會清除所有路徑點。
12. 點擊「清除全部範圍」會清除所有範圍點與已產生的區域路徑。
13. 點擊「清除目的地」會清除所有慢走目的地。
14. 地圖不會因為位置更新自動跳回目前位置，只有點擊「回到目前位置」或開啟「跟隨目前位置」時才會移動視角。

## 使用路徑巡航

1. 切換到「路徑選點」，在地圖上點選至少 2 個路徑點。
2. 設定路徑巡航速度，範圍 1 到 100 km/h。
3. 選擇路徑起點模式，預設為「從最近點開始」。
4. 選擇結束方式，預設為「折返」。
5. 點擊「開始路徑巡航」。
6. 應用程式會依序沿著座標點移動，並每秒推送一次模擬定位。
7. 點擊「停止巡航」可停止送出巡航位置。
8. 巡航中刪除路徑點時，如果剩餘點數不足 2 個會自動停止；若仍有足夠點數，會從目前位置附近的點續行。

兩種起點模式：

- 「從最近點開始」：從目前模擬位置附近的路徑點開始巡航。
- 「從第一點開始」：固定先朝第 1 個路徑點移動，再依序巡航。

三種結束方式：

- 「停止」：到達最後一個座標點後停止。
- 「循環」：到達最後一個座標點後回到第一個座標點繼續巡航。
- 「折返」：到達最後一個座標點後反向走回去，再來回巡航。

## 使用區域巡航

1. 切換到「區域圈選」，在地圖上點選至少 3 個範圍點，形成一個多邊形範圍。
2. 選擇區域產生方式：「隨機巡航」或「蛇形掃描」。
3. 點擊「產生區域路徑」。
4. 「隨機巡航」會在多邊形範圍的外接矩形中隨機取點，並用多邊形內點檢查確認點是否位於範圍內。
5. 「蛇形掃描」會用固定掃描線在多邊形範圍內產生可重現的蛇形路徑。
6. 「隨機巡航」會產生 10 到 50 個區域座標點，並用最近鄰排序降低路徑亂跳。
7. 地圖會顯示產生的座標點與折線。
8. 設定區域巡航速度，範圍 1 到 100 km/h。
9. 如果範圍點少於 3 個，「產生區域路徑」會停用。
10. 點擊「開始區域巡航」後，應用程式會沿著產生路徑移動。
11. 刪除範圍點後，已產生的區域路徑會被清除；如果正在區域巡航，會自動停止。
12. 區域巡航預設使用「折返」模式。

## 使用慢走到目的地

1. 進入「地圖繞行模式」分頁。
2. 點擊「新增目的地」。
3. 在地圖上點選一個或多個目的地。
4. 設定「慢走速度」，範圍 1 到 100 km/h，預設 5 km/h。
5. 點擊「開始慢走」後，應用程式會從目前模擬位置開始，依序慢慢走到每個目的地。
6. 慢走每秒更新一次位置，會依照目前位置與下一個目的地計算方向角，不會瞬移。
7. 畫面會顯示目前目標點、剩餘距離與預估到達時間。
8. 點擊「停止慢走」會停止移動，並停留在最後模擬位置。
9. 所有目的地都抵達後，會自動進入「停留在最後位置」。
10. 若要清掉目的地，請點擊「清除目的地」。

請只用這些功能測試你自己的定位應用程式或你有權測試的環境。不得用於第三方應用程式、遊戲作弊、規避偵測或違反服務條款。

若目標應用程式明確拒絕模擬定位，本工具不提供繞過偵測功能，也不提供規避平台或服務條款限制的功能。

本工具不實作隱藏模擬定位、root、hook、注入、控制第三方應用程式介面，或任何繞過偵測功能。

## 打包 APK 與 ADB 安裝

APK 產出路徑：

```text
app/build/outputs/apk/debug/app-debug.apk
```

Windows PowerShell：

```powershell
./gradlew.bat assembleDebug
adb devices
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

若 PowerShell 找不到 Android SDK，或系統 Java 版本不適合 Android Gradle Plugin，可先設定：

```powershell
$env:JAVA_HOME="C:\Program Files\Android\Android Studio\jbr"
$env:ANDROID_HOME="$env:LOCALAPPDATA\Android\Sdk"
$env:ANDROID_SDK_ROOT="$env:LOCALAPPDATA\Android\Sdk"
$env:Path += ";$env:LOCALAPPDATA\Android\Sdk\platform-tools"
```

Linux/macOS：

```bash
./gradlew assembleDebug
adb devices
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

補充事項：

- 手機需開啟 USB 偵錯。
- 第一次連線要在手機上允許 RSA 授權。
- `adb devices` 要看到 `device` 狀態才可安裝。
- 若出現 `unauthorized`，要重新插拔 USB 並在手機上允許。
- 若已有舊版 APK，使用 `adb install -r` 覆蓋安裝。
- 若安裝時出現 `INSTALL_FAILED_USER_RESTRICTED`，請在手機開啟「透過 USB 安裝」或同品牌對應的安全開關，並在手機彈窗中允許安裝。
- 如果沒有 `gradlew`，可以用 Android Studio 開啟專案後：`Build > Build Bundle(s) / APK(s) > Build APK(s)`。
